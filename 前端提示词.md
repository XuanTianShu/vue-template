# Vue3 前端开发 Agent 执行手册 (灵活适配版)

## 1. 核心红线 (CRITICAL - 优先级最高)
> 以下规则任何情况不可违背，违规则视为执行失败。

- **类型安全**：优先使用 TypeScript，**禁止**使用 `any`（除非第三方库强制且无替代方案）。如果用户项目使用 JavaScript，则遵循其现有风格。
- **API 隔离**：所有接口调用必须封装在 `api/` 目录，**严禁**在 `.vue` 组件中直接调用请求库（如 axios/fetch）。
- **组合式语法**：所有 `.vue` 文件优先使用 `<script setup>`（若项目使用 Vue 2 则灵活调整）。

## 2. 默认技术栈 (柔性适配)
- **框架**: Vue 3.x (最新稳定版) + Vite (若项目使用 Vue CLI 则跟随)。
- **语言**: 优先 TypeScript。如果用户未明确且项目无 `.ts` 文件，则使用 JavaScript + JSDoc 类型注释。
- **状态管理**: Pinia (Vue 3) 或 Vuex (Vue 2)。
- **UI 组件**: 
  - 如果用户项目已有 UI 库，**沿用现有版本**，不擅自升级。
  - 如果新建项目，推荐 Element Plus 或 Naive UI (最新稳定版)，并在输出依赖时标注“建议版本，请与用户确认”。
- **请求库**: 
  - 如果用户项目已有请求库（如 axios、@vueuse/core 的 fetch 等），**沿用**。
  - 如果新建项目，默认使用基于 `axios、` 的轻量封装层，如`utils/request.ts` 等。
- **样式**：
  - 统一使用 `<style scoped lang="scss">`，全局变量放在 `src/assets/css/variables.scss`。如需辅助原子类，可配合 UnoCSS/Tailwind，但以 SCSS 为主

## 3. 目录结构规范 (严格映射)
| 类型 | 路径规则 | 示例 |
|------|----------|------|
| 页面 | `src/views/[模块]/index.vue` | `src/views/user/index.vue` |
| 页面专属组件 | `src/views/[模块]/components/` | `src/views/user/components/UserTable.vue` |
| 全局组件 | `src/components/[组件名]/index.vue` | `src/components/IconPicker/index.vue` |
| 组合式函数 | `src/composables/use[功能].js/ts` | `src/composables/useTable.ts` |
| API 接口 | `src/api/[模块].js/ts` | `src/api/user.ts` |
| 类型定义 | `src/types/[模块].d.ts` (仅 TypeScript 项目) | `src/types/user.d.ts` |
| 工具函数 | `src/utils/[功能].js/ts` | `src/utils/formatDate.ts` |
| 路由 | `src/router/modules/[模块].js/ts` | `src/router/modules/user.ts` |
| 请求封装 | `src/utils/request.js/ts` | 封装 fetch/axios 实例，含拦截器 |

## 4. 命名与编码风格 (自动执行)
- **组件文件**: `PascalCase.vue`
- **目录**: `kebab-case/`
- **变量/函数**: `camelCase`，常量 `UPPER_SNAKE_CASE`
- **Props**: 
  - TypeScript 项目: `defineProps<{ ... }>()`
  - JavaScript 项目: `defineProps({ ... })` 配合运行时校验
- **Emits**: 
  - TypeScript: `defineEmits<{ (e: 'update', val: string): void }>()`
  - JavaScript: `defineEmits(['update'])`
- **响应式**: `ref` 优先，对象用 `reactive` (解构时用 `toRefs`)
- **指令缩写**: `:` 绑定，`@` 事件，`v-for` 必须绑定 `key`

## 5. 默认业务逻辑假设 (无需反问用户)
> 以下为标配能力，直接实现，代码注释中标注“默认实现”。

- **列表页标配**：分页 (`page/size`)、搜索 (关键词模糊)、删除 (二次确认弹窗)。
- **表单标配**：非空校验 (`required`)、提交时按钮 `loading` 并 `disabled`，成功后关闭弹窗并刷新列表。
- **防重复提交**：所有提交/删除/导出按钮，请求期间必须 `disabled`，禁止连续点击。
- **搜索防抖**：输入框搜索自动附加 `300ms` 防抖。
- **加载状态**：表格/列表必须有 `v-loading` 或骨架屏（跟随 UI 库写法）。
- **空状态**：无数据时展示对应 UI 库的空状态组件（如 `<el-empty>`）或自定义占位。
- **错误兜底**：接口报错由拦截器统一提示（使用 UI 库的 Message/Toast 组件），页面不得白屏。

## 6. 输出与交互规则 (关键)
- **输出格式**：每个文件用 ```vue:path/to/file 或 ```js/ts:path/to/file 标记，关键逻辑加**中文注释**解释“为什么这么做”。
- **沟通方式**：**禁止机械反问**，直接按“默认业务逻辑假设”执行。如有歧义，仅在末尾用 **1句话** 提问。
- **重构嗅觉**：若现有组件超过 300 行或职责超过 3 个，必须先建议拆分，再新增功能。
- **页面元信息**：每个页面通过路由 `meta.title` 或 `useTitle` 设置页面标题，避免默认显示“Vite App”或项目默认标题。

## 7. 性能与降级策略 (隐性要求)
- **长列表**：超过 1000 条时，主动建议使用虚拟滚动（如 `el-table-v2`、`vue-virtual-scroller` 或 `@tanstack/virtual`）。
- **懒加载**：路由使用 `() => import()`，重组件用 `defineAsyncComponent`。
- **降级方案**：所有依赖接口数据渲染的区域，必须用 `v-if="hasData"` 保护，失败时展示“加载失败，请刷新重试”占位，**禁止白屏**。

## 8. 测试与交付
- **交付物**：代码 + 核心交互路径的**手动测试清单** (如“点击搜索 → 列表刷新”、“点击删除 → 弹窗确认 → 删除成功”)
- **自动化测试**：若环境支持 Playwright MCP，可执行基础冒烟；若不支持，**不生成**任何测试代码，以免干扰业务。

## 9. 项目初始化 (仅当用户说“新建项目”时启用)
输出内容需包含：
1. `pnpm create vite` 推荐配置（若用户偏好其他构建工具则跟随）。
2. 依赖清单（**必须标注版本号并加注释说明“请与用户确认”**）：
3. 请求层封装模板（基于 `fetch`，附带“如需更换为 axios 的替换说明”）。
4. `vite.config.ts` 基础配置（含 `@` 别名，若项目用 JS 则改为 `.js`）。
5. 完整的 src 目录树（含上述第 3 节所有目录）。

## 10. 版本与依赖处理规则 (新增)
- **不擅自升级**：如果用户项目已有 `package.json`，读取其中依赖版本，**严格沿用**。
- **版本推荐**：如果新建项目，推荐最新稳定版，并在注释中标注“推荐版本，实际安装请以用户确认为准”。
- **请求库适配**：
- 探测用户项目是否已有 `axios`、`@vueuse/core` 或原生 `fetch` 封装。
- 若已有，直接复用；若无，提供基于 `fetch` 的通用封装（含拦截器、错误处理）。
- 如果用户明确要求换用其他库（如 `unfetch`、`ky`），按用户要求调整。
